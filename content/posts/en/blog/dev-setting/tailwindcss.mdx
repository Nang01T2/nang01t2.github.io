Tailwind CSS is a utility-first CSS framework that lets you style your elements using pre-defined classes. It is a great way to create responsive and customizable designs without writing a lot of CSS code. In this post, I will show you how to set up Tailwind CSS with Next.js, a React framework that provides features like server-side rendering, static site generation, and code splitting.

## Introduction to Tailwind CSS

Tailwind CSS is based on the idea of composing your styles using utility classes that represent common CSS properties and values. For example, instead of writing this CSS code:

```css
.button {
  display: inline-block;
  padding: 0.5rem 1rem;
  border: none;
  border-radius: 0.25rem;
  background-color: blue;
  color: white;
  font-weight: bold;
}
```

You can use these Tailwind classes:

```html
<button class="inline-block px-4 py-2 border-none rounded bg-blue-500 text-white font-bold">
  Click me
</button>
```

Tailwind provides hundreds of utility classes for spacing, colors, typography, layout, flexbox, grid, and more. You can also customize the default theme, add your own classes, and use plugins to extend the functionality of Tailwind.

## Basic setup of Tailwind CSS with Next.js

To use Tailwind CSS with Next.js, you need to install some packages and configure some files. Here are the steps:

1. Create a new Next.js project using `create-next-app`. You can also use TypeScript and ESLint if you want.

```bash
npx create-next-app@latest my-project --typescript --eslint
cd my-project
```

2. Install Tailwind CSS and its peer dependencies via npm.

```bash
npm install -D tailwindcss postcss autoprefixer
```

3. Run the init command to generate both tailwind.config.js and postcss.config.js.

```bash
npx tailwindcss init -p
```

4. Add the paths to all of your template files in your tailwind.config.js file. This will help Tailwind to remove unused classes and optimize the final CSS bundle.

```ts title="tailwind.config.js"
module.exports = {
  content: [
    "./app/**/*. {js,ts,jsx,tsx,mdx}",
    "./pages/**/*. {js,ts,jsx,tsx,mdx}",
    "./components/**/*. {js,ts,jsx,tsx,mdx}",
    // Or if using `src` directory:
    // "./src/**/*. {js,ts,jsx,tsx,mdx}",
  ],
  theme: {
    extend: {},
  },
  plugins: [],
};
```

5. Add the `@tailwind` directives for each of Tailwind's layers to your globals.css file.

```ts title="global.css"
@tailwind base;
@tailwind components;
@tailwind utilities;
```

6. Import the `globals.css` stylesheet in your root layout component (e.g. app/layout.tsx) to apply the styles to every route in your application.

```ts title="app/layout.tsx"
import type { Metadata } from "next";

// These styles apply to every route in the application
import "./globals.css";

export const metadata: Metadata = {
  title: "Create Next App",
  description: "Generated by create next app",
};

export default function RootLayout({
children,
}: {
children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );
}
```

7. Run your development server with npm run dev

```bash
npm run dev
```

8. Start using Tailwind's utility classes to style your content.

## Setup more plugins

Tailwind CSS has a rich ecosystem of plugins that add more features and utilities to the framework. In this section, I will show you how to install and use two popular plugins: prettier-plugin-tailwindcss and eslint-plugin-tailwindcss.

### prettier-plugin-tailwindcss

This plugin formats your HTML code with Tailwind classes using Prettier. It helps you to keep your code consistent and readable by sorting and grouping the classes according to their categories.

To install this plugin, run:

```bash
npm install -D prettier-plugin-tailwindcss
```

Then add this line to your .prettierrc file:

```json
{
  "plugins": ["prettier-plugin-tailwindcss"]
}
```

Now you can format your code with Prettier as usual. For example:

```html
<!-- Before -->
<button class="bg-blue-500 text-white font-bold px-4 py-2 rounded inline-block border-none">
  Click me
</button>

<!-- After -->
<button class="inline-block px-4 py-2 border-none rounded bg-blue-500 text-white font-bold">
  Click me
</button>
```

### eslint-plugin-tailwindcss

This plugin checks your HTML code with Tailwind classes using ESLint. It helps you to avoid typos and errors by reporting invalid or unknown classes.

To install this plugin, run:

```bash
npm install -D eslint-plugin-tailwindcss
```

Then add this line to your .eslintrc.json file:

```json
{
  "plugins": ["tailwindcss"],
  "rules": {
    "tailwindcss/classnames-order": "warn",
    "tailwindcss/no-custom-classname": "warn",
    "tailwindcss/no-contradicting-classname": "error"
  }
}
```

Now you can lint your code with ESLint as usual. For example:

```html
<!-- Invalid class name -->
<button class="bg-blu-500">Click me</button>
<!-- ESLint error: Unknown class name 'bg-blu-500'. 
Did you mean 'bg-blue-500'? (tailwindcss/no-custom-classname) -->

<!-- Contradicting class name -->
<button class="bg-blue-500 bg-red-500">Click me</button>
<!-- 
ESLint error: Conflicting class name 'bg-red-500'. 
Class 'bg-blue-500' should be removed or replaced. 
(tailwindcss/no-contradicting-classname) 
-->
```

## Using tailwind-merge
Tailwind CSS has a built-in feature called JIT mode that allows you to generate **custom classes on demand** using brackets syntax. For example:

```html
<button class="bg-[#1da1f2] text-white">Click me</button>
```

However, sometimes you may want to use variables or expressions instead of hard-coded values in your brackets syntax. For example:

```html
// Using a variable for the background color
const color = "#1da1f2";
<button class={`bg-[${color}] text-white`}>Click me</button>

// Using an expression for the font size
const size = 3;
<button class={`text-[${size}xl] font-bold`}>Click me</button>
```

Unfortunately, JIT mode does not support variables or expressions in brackets syntax yet. This means that the above examples will not work as expected.

To solve this problem, you can use a library called [tailwind-merge](https://github.com/lukejacksonn/tailwind-merge). This library allows you to merge multiple Tailwind classes into one using a simple function.

To install this library, run:

```bash
npm install tailwind-merge
```

Then import it in your component file:

```ts
import tw from "tailwind-merge";
```

Now you can use it to merge any Tailwind classes into one using brackets syntax with variables or expressions. For example:

```html
// Using a variable for the background color
const color = "#1da1f2";
<button className={tw`bg-[${color}] text-white`}>Click me</button>

// Using an expression for the font size
const size = 3;
<button className={tw`text-[${size}xl] font-bold`}>Click me</button>
```

The library will generate a unique class name for each combination of classes and inject it into the final CSS bundle.

## Setup fonts of tailwindcss
Tailwind CSS comes with a default font stack that includes system fonts for different platforms. However, you may want to use custom fonts for your project.

To use custom fonts with Tailwind CSS, you need to do two things:

1. Import or load your fonts in your HTML document or `global` stylesheet.
2. Define your custom font family in your `tailwind.config.js` file.

For example, let's say you want to use [Inter](https://fonts.google.com/specimen/Inter) as your primary font and [Roboto Mono](https://fonts.google.com/specimen/Roboto+Mono) as your secondary font.

First, import or load these fonts in your HTML document or global stylesheet using `<link>` or `@import`. For example:

```html
<!-- Importing fonts from Google Fonts -->
<link
  rel="stylesheet"
  href="https://fonts.googleapis.com/css?family=Inter:400,700|Roboto+Mono:400,700&display=swap"
/>
```

```css
<!-- Or loading fonts from local files -->
<style>
@font-face {
  font-family: "Inter";
  font-style: normal;
  font-weight: 400;
  src: url("/fonts/Inter-Regular.woff2") format("woff2");
}
@font-face {
  font-family: "Inter";
  font-style: normal;
  font-weight: 700;
  src: url("/fonts/Inter-Bold.woff2") format("woff2");
}
@font-face {
  font-family: "Roboto Mono";
  font-style: normal;
  font-weight: 400;
  src: url("/fonts/RobotoMono-Regular.woff2") format("woff2");
}
@font-face {
  font-family: "Roboto Mono";
  font-style: normal;
  font-weight: 700;
  src: url("/fonts/RobotoMono-Bold.woff2") format("woff2");
}
</style>
```

Second, define your custom font family in your `tailwind.config.js` file. You can either add a new font family or replace an existing one. For example:

```js title="tailwind.config.js"
module.exports = {
  theme: {
    fontFamily: {
      // Adding a new font family
      inter: ["Inter", "sans-serif"],
      // Replacing the default sans-serif font family
      sans: ["Roboto Mono", "monospace"],
    },
  },
};
```

Now you can use your custom fonts with the font-family utilities. For example:

```html
<h1 class="text-4xl font-inter">This is Inter</h1>
<p class="text-lg font-sans">This is Roboto Mono</p>
```
